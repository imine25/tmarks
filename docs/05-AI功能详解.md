# AI 功能详解

## AI 提供商支持

### 支持的提供商

TMarks 支持 8+ AI 提供商，用户可以自由选择：

1. **OpenAI**
   - 模型：gpt-4o, gpt-4o-mini, gpt-3.5-turbo
   - API URL：https://api.openai.com/v1

2. **Anthropic (Claude)**
   - 模型：claude-3-5-sonnet, claude-3-opus, claude-3-haiku
   - API URL：https://api.anthropic.com/v1

3. **DeepSeek**
   - 模型：deepseek-chat, deepseek-coder
   - API URL：https://api.deepseek.com/v1

4. **智谱 AI (GLM)**
   - 模型：glm-4, glm-4-flash
   - API URL：https://open.bigmodel.cn/api/paas/v4

5. **ModelScope**
   - 模型：qwen-turbo, qwen-plus, qwen-max
   - API URL：https://dashscope.aliyuncs.com/compatible-mode/v1

6. **SiliconFlow**
   - 模型：Qwen/Qwen2.5-7B-Instruct
   - API URL：https://api.siliconflow.cn/v1

7. **iFlow**
   - 模型：deepseek-ai/DeepSeek-V3
   - API URL：https://api.iflow.cn/v1

8. **自定义提供商**
   - 支持任何兼容 OpenAI API 格式的提供商
   - 自定义 API URL 和模型名称

### 配置方式

#### 配置项说明
- **provider**：提供商名称（openai, anthropic, deepseek 等）
- **apiKey**：API 密钥
- **apiUrl**：自定义 API 地址（可选）
- **model**：模型名称（可选）
- **temperature**：温度参数，控制随机性（0-1）
- **maxTokens**：最大生成 Token 数

#### 配置存储
- 保存在 Chrome Storage Sync
- 跨设备同步
- 加密存储敏感信息

#### 配置验证
- 测试 API 连接
- 验证 API Key 有效性
- 检查模型可用性

## AI 标签推荐

### 功能概述

在保存书签时，AI 会根据页面内容自动推荐相关标签，帮助用户快速分类。

### 推荐流程

```
1. 提取页面信息
   - 标题
   - URL
   - Meta 描述
   - 页面内容（前 500 字）
   ↓
2. 构建 AI Prompt
   - 系统提示词
   - 页面信息
   - 现有标签列表（可选）
   ↓
3. 调用 AI API
   - 发送请求
   - 等待响应
   - 解析 JSON 结果
   ↓
4. 返回推荐标签
   - 标签名称
   - 推荐理由
   - 置信度
```

### Prompt 模板

#### 系统提示词结构
- 角色定义：专业的书签标签推荐助手
- 任务说明：根据网页信息推荐标签
- 要求列表：
  - 标签简洁明了（2-4 个字）
  - 优先使用现有标签
  - 准确描述网页主题
  - 返回 JSON 格式

#### 输入信息
- 现有标签列表
- 网页标题
- 网页 URL
- Meta 描述
- 页面内容（前 500 字）

#### 输出格式
- JSON 格式
- 包含标签数组
- 3-5 个推荐标签

### 自定义 Prompt

#### 配置选项
- **enableCustomPrompt**：是否启用自定义 Prompt
- **customPrompt**：自定义的 Prompt 模板
- **variables**：可用的变量列表

#### 支持的变量
- `{{title}}` - 网页标题
- `{{url}}` - 网页 URL
- `{{description}}` - 网页描述
- `{{content}}` - 页面内容
- `{{existingTags}}` - 现有标签列表

#### 变量替换
- 使用双花括号语法
- 自动替换为实际值
- 支持数组类型（如 existingTags）

#### 使用场景
- 调整推荐风格
- 添加特定要求
- 适配不同语言
- 优化推荐质量

### 降级策略

#### 降级流程
1. 尝试调用 AI API 推荐标签
2. 如果失败，切换到关键词匹配
3. 记录警告日志
4. 返回匹配结果

#### 关键词匹配逻辑
1. 从页面内容提取关键词
2. 获取用户现有标签列表
3. 匹配包含关键词的标签
4. 返回匹配的标签

#### 降级触发条件
- AI API 请求失败
- 网络连接错误
- API Key 无效
- 响应超时
- 返回格式错误

#### 用户体验
- 降级过程透明
- 功能保持可用
- 可选显示提示
- 不影响保存流程

## AI 文件夹推荐

### 功能概述

在 NewTab 保存书签时，AI 会根据书签内容推荐合适的文件夹位置。

### 推荐流程

```
1. 获取现有文件夹结构
   - 遍历 TMarks 根目录
   - 提取所有文件夹路径
   ↓
2. 构建 AI Prompt
   - 书签信息（标题、URL、描述）
   - 现有文件夹列表
   - 推荐数量限制
   ↓
3. 调用 AI API
   - 发送请求
   - 等待响应
   - 解析 JSON 结果
   ↓
4. 返回推荐文件夹
   - 文件夹路径
   - 推荐理由
   - 置信度
```

### Prompt 模板

#### 系统提示词结构
- 角色定义：专业的书签分类助手
- 任务说明：从现有文件夹推荐位置
- 要求列表：
  - 推荐指定数量的文件夹
  - 优先语义相关的文件夹
  - 考虑 URL 域名特征
  - 返回 JSON 格式

#### 输入信息
- 现有文件夹列表（带编号）
- 书签标题
- 书签 URL
- 书签描述

#### 输出格式
- JSON 格式
- 包含文件夹路径数组
- 推荐数量可配置（10-20 个）

### 配置选项

#### 基本配置
- **enabled**：是否启用文件夹推荐
- **count**：推荐数量（10-20 个）

#### 高级配置
- **enableCustomPrompt**：是否使用自定义 Prompt
- **customPrompt**：自定义 Prompt 模板

#### 推荐策略
- 语义匹配优先
- 考虑域名特征
- 参考历史分类
- 支持多语言

#### 使用场景
- NewTab 保存书签时
- 快速选择文件夹位置
- 减少手动查找时间
- 提高分类准确性

## AI 智能整理

### 功能概述

AI 智能整理是 TMarks 最强大的功能之一，可以自动分析所有书签，按照语义和使用频率重新组织文件夹结构。

### 整理流程

```
1. 数据收集阶段
   ├─ 收集所有浏览器书签
   ├─ 按域名聚合统计
   ├─ 加载快捷方式使用数据
   └─ [可选] 读取历史记录
   ↓
2. 数据分析阶段
   ├─ 计算域名访问频率
   ├─ 计算最近访问时间
   ├─ 生成热度评分
   └─ 提取原始路径分布
   ↓
3. 生成域名摘要
   ├─ 域名基本信息
   ├─ 热度评分
   ├─ 原始路径
   └─ 示例书签
   ↓
4. AI 分析阶段
   ├─ 输入域名摘要
   ├─ 输入用户规则
   ├─ 输入现有文件夹
   └─ 输出整理方案
   ↓
5. 执行整理阶段
   ├─ 清空工作区
   ├─ 创建文件夹结构
   └─ 按方案复制书签
```

### 数据收集

#### 域名摘要结构
- **domain**：域名
- **count**：书签数量
- **heatScore**：热度评分（0-100）
- **recencyScore**：最近访问评分（0-100）
- **originalPaths**：原始路径列表
- **examples**：示例书签（标题、URL、路径）

#### 收集流程
1. 收集所有浏览器书签
2. 按域名聚合统计
3. 加载快捷方式使用数据
4. 可选读取历史记录（最近 N 天）
5. 生成域名摘要列表

#### 数据来源
- 浏览器书签 API
- NewTab 使用统计
- 浏览器历史记录（可选）
- 现有文件夹结构

### 热度计算

#### 评分算法
热度评分由三部分组成（总分 100）：

**1. 快捷方式使用频率（权重 40%）**
- 统计域名在 NewTab 的点击次数
- 与最高使用次数对比
- 计算相对得分

**2. 历史访问频率（权重 40%）**
- 统计最近 N 天的访问次数
- 与最高访问次数对比
- 计算相对得分

**3. 最近访问时间（权重 20%）**
- 计算距离最后访问的天数
- 每天衰减 5 分
- 最低 0 分

#### 评分范围
- 最低：0 分
- 最高：100 分
- 用于排序和优先级判断

### AI Prompt

#### 系统提示词结构

**角色定义**
- 专业的书签整理助手

**整理原则**
1. **聚合原则**
   - 同产品不同子域名合并
   - 功能相似的网站合并
   - 少于 3 个域名的分类合并到更大类别

2. **层级规则**
   - 只允许 1-2 层文件夹
   - 禁止三层嵌套
   - 一级目录数量限制（可配置）

3. **优先级**
   - 用户自定义规则（最高优先级）
   - 历史访问热度（Top N 放一级目录）
   - 现有文件夹路径（优先保留）
   - AI 语义分析

4. **特殊要求**（根据配置）
   - 是否禁止新建一级目录
   - 是否禁止新建文件夹
   - 是否优先保留原路径

#### 输入信息
- 用户自定义规则
- 现有文件夹列表
- 域名统计（按热度排序）
  - 域名、书签数量
  - 热度评分、最近访问评分
  - 原始路径、示例书签

#### 输出格式
- JSON 格式
- domainMoves 对象：域名 → 文件夹路径映射

### 配置选项

#### 历史热度分析
- **enableHistoryHeat**：是否启用历史热度分析
- **historyDays**：分析天数（7-90 天）
- **historyHeatTopN**：Top N 高频域名放一级目录（10-50）

#### 层级控制
- **topLevelCount**：一级目录数量限制（2-7）
- **strictHierarchy**：是否禁止新建一级目录
- **allowNewFolders**：是否允许新建文件夹
- **preferOriginalPaths**：是否优先保留原路径

#### 自定义规则
- **rules**：用户自定义分类规则（文本）
  - 示例："GitHub 相关放到「开发」"
  - 示例："设计工具放到「设计」"

#### 自定义 Prompt
- **enableCustomPrompt**：是否使用自定义 Prompt
- **customPrompt**：自定义 Prompt 模板

### 执行整理

#### 整理流程

**1. 清空工作区**
- 获取 TMarks 根文件夹 ID
- 删除所有子文件夹和书签
- 原始书签保留在浏览器其他位置

**2. 创建文件夹结构**
- 遍历 AI 规划的所有路径
- 按路径层级创建文件夹
- 使用 Map 记录路径到 ID 的映射
- 避免重复创建

**3. 复制书签**
- 遍历所有原始书签
- 提取域名
- 查找 AI 规划的目标路径
- 在目标文件夹创建书签副本

#### 错误处理
- TMarks 文件夹不存在时抛出错误
- 创建文件夹失败时记录日志
- 复制书签失败时跳过该书签
- 保证部分成功不影响整体

### 进度监控

#### 进度数据结构
- **stage**：当前阶段（collecting, analyzing, executing, done）
- **progress**：进度百分比（0-100）
- **message**：当前操作描述
- **logs**：日志列表
  - level：日志级别（info, warn, error, success）
  - message：日志内容
  - timestamp：时间戳

#### 进度报告器
- 记录所有操作日志
- 发送进度更新到 UI
- 支持四种日志级别
- 实时显示当前状态

#### UI 显示
- 实时日志控制台
- 分级日志（不同颜色）
- 进度条显示
- 支持复制日志
- 自动滚动到底部

#### 使用场景
- 数据收集阶段
- AI 分析阶段
- 执行整理阶段
- 完成后总结

## AI 批量整理（Web 应用）

### 功能概述

在 Web 应用中导入书签时，可以使用 AI 批量整理功能，自动为书签生成标签和优化描述。

### 整理流程

```
1. 上传书签文件（HTML/JSON）
   ↓
2. 解析书签数据
   ↓
3. [可选] AI 批量整理
   - 分析书签内容
   - 生成智能标签
   - 优化描述信息
   ↓
4. 预览整理结果
   ↓
5. 确认导入
   ↓
6. 批量保存到数据库
```

### Prompt 模板

#### 系统提示词结构
- 角色定义：专业的书签整理助手
- 任务说明：为书签生成标签和优化描述
- 要求列表：
  - 每个书签生成 2-4 个标签
  - 标签简洁明了（2-4 个字）
  - 优化描述信息
  - 返回 JSON 格式

#### 输入信息
- 书签列表（带编号）
  - 标题
  - URL
  - 原始描述

#### 输出格式
- JSON 格式
- results 数组
  - index：书签索引
  - tags：标签数组
  - description：优化后的描述

### 批量处理

#### 批量策略
- 每批处理 50 个书签
- 避免单次请求过大
- 提高处理成功率

#### 处理流程
1. 将书签列表分批
2. 逐批调用 AI API
3. 收集所有结果
4. 处理失败的批次

#### 错误处理
- 单批失败不影响其他批次
- 失败时使用降级策略：
  - 保留原始标签
  - 保留原始描述
- 记录失败日志

#### 性能优化
- 控制并发数量
- 避免 API 限流
- 显示处理进度

## 性能优化

### 请求缓存

#### 缓存机制
- 使用 Map 存储 AI 响应
- 缓存键：Prompt 哈希 + 配置哈希
- 避免重复请求

#### 缓存策略
- 相同 Prompt 直接返回缓存
- 配置变化时失效
- 内存缓存，重启清空

#### 适用场景
- 重复的标签推荐
- 相同的文件夹推荐
- 批量处理中的重复内容

#### 性能提升
- 减少 API 调用次数
- 降低请求成本
- 提高响应速度

### 并发控制

#### 并发限制器
- 限制同时进行的 AI 请求数量
- 默认最大并发数：3
- 避免触发 API 限流

#### 工作原理
1. 维护运行中的请求计数
2. 达到上限时等待
3. 请求完成后释放槽位
4. 自动调度等待的请求

#### 使用场景
- 批量标签推荐
- 批量文件夹推荐
- 批量书签整理

#### 优势
- 避免 API 限流
- 控制资源使用
- 提高稳定性

### 超时控制

#### 超时机制
- 默认超时时间：30 秒
- 使用 Promise.race 实现
- 超时后抛出错误

#### 超时处理
- 取消当前请求
- 返回超时错误
- 触发降级策略

#### 配置选项
- 可自定义超时时间
- 不同场景不同超时
  - 标签推荐：30 秒
  - 文件夹推荐：30 秒
  - 智能整理：60 秒

#### 用户体验
- 避免长时间等待
- 及时反馈错误
- 提供重试选项

## 错误处理

### 常见错误

#### 错误类型枚举
- **NETWORK_ERROR**：网络连接失败
- **API_KEY_INVALID**：API Key 无效（401）
- **RATE_LIMIT**：请求频率超限（429）
- **TIMEOUT**：请求超时
- **PARSE_ERROR**：响应解析失败
- **UNKNOWN**：未知错误

#### 错误识别
- 根据错误消息判断类型
- 根据 HTTP 状态码判断
- 返回对应的错误类型

#### 错误处理策略
- 网络错误：重试
- API Key 无效：提示用户检查配置
- 限流：延迟后重试
- 超时：提示用户重试
- 解析错误：记录日志并降级

### 用户提示

#### 错误消息映射
- **网络连接失败**：请检查网络设置
- **API Key 无效**：请检查配置
- **请求频率超限**：请稍后重试
- **请求超时**：请重试
- **响应解析失败**：请重试
- **服务异常**：请稍后重试

#### 提示策略
- 使用友好的中文提示
- 提供具体的解决建议
- 避免技术术语
- 支持多语言

#### 显示方式
- Toast 通知
- 错误对话框
- 内联错误提示
- 日志记录
