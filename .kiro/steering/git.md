# RULES.md（协作执行规范）

## 1. 代码变更规模

* 单次任务 / 单次回复对应的实现：**新增代码 ≤ 300 行**
* 如确需超出：**拆分为多个小改动 / 多次提交**，且每次仍需满足 **≤ 300 行**

## 2. Git 提交提示（强制）

* 每次**完成代码修改或功能实现**后，必须在回复末尾追加一段**可复制的中文提交信息**
* 该段内容**只包含提交信息本身**，格式如下：

```bash
这里写提交信息（中文、准确、动词开头优先）
```

## 3. 文档创建（默认禁止）

* **未被明确要求时：禁止创建**任何总结 / 报告 / 临时文档

### 3.1 禁止清单（示例）

* `SUMMARY.md`
* `REPORT.md`
* `AUDIT_REPORT.md`
* `BUG_FIX_REPORT.md`
* `TEMP.md`
* `NOTES.md`
* 任何以 `_REPORT` / `_SUMMARY` / `_AUDIT` 结尾的文件

### 3.2 允许创建

* 用户明确要求的文档
* 项目必需的配置或基础文档（如 `README.md`、`package.json` 等）
* 必要的代码文件与测试文件
* 经用户确认的正式项目文档

## 4. 输出风格

* **直接交付结果**：只给必要步骤与关键变更点
* **避免冗余总结**：除非用户明确要求
* 若规则存在冲突或需要新增文档：**优先按更严格约束执行**（先不建文档、先拆分改动）
